SPRING CAMP 2018

스프링캠프에서는 JVM(Java Virtual Machine) 기반 시스템의 백엔드(Back-end) 또는 서버사이드(Server-side)라고 칭하는 영역을 개발하는 애플리케이션 서버 개발에 관한 기술과 정보, 경험을 공유하는 컨퍼런스입니다.

핵심주제로 Micro Service Architecture와 Cloud Native를 다루고 있으며, 그외 Architecture나 JVM Language, Software Development Process 등 애플리케이션 서버 개발에 필요한 다양한 주제를 다루려고 노력하고 있습니다.

Consumer Driven Contract 기법을 활용한 마이크로서비스 API의 진화

마이크로서비스가 본격적으로 확산이되면 API 제공자와 사용자간의 dependency가 생겨서 마이크로서비스 API의 개선과정이 매우 복잡해지고 느려지게됩니다. 이 문제를 해결하기 위해 Consumer Driven Contract기법을 적용한 Spring cloud contract 프로젝트의 활용방법에 대해 소개하고 샘플 애플리케이션을 통해 개발자들이 API를 개발하고 테스트하며 개선하는 방식에 대해 알아봅니다.

MicroProfile (for Enterprise Java Microservices)

김명주

MicroProfile (for Enterprise Java Microservices)

MicroProfile은 Enterprise Java 환경에 최적화된 Microservices 개발을 지원하는 Eclipse 재단의 opensource project 입니다. Java EE의 더딘 발전을 안타까워 하던 개발자, 그룹, 벤더 들에 의해 2016년 탄생하게 되었습니다. 이들은 언젠가 MicroProfile이 Java EE 표준으로 자리매김 하기를 기대하고 있습니다.

본 발표에서는 MicroProfile의 개요, 아키텍처, 주요기능(Config, Fault Tolerance, Health Check, Metrics, JWT Authentication, OpenAPI, OpenTracing, Rest Client)과 개발방법(데모), 앞으로의 발전방향에 대해 소개할 예정입니다.

빠르게 티켓 구매하는 방법

티켓을 빠르게 구매하시기 위해서는 사전에 쿠팡 회원 가입 후 티켓 판매 시작 전에 로그인을 해두시면 좋습니다.
또한 가장 빠른 티켓 구매 방법은 로켓페이를 이용하는 것으로 사전에 로켓페이에 결제 방식을 등록해 두시면 좋습니다.

소진된 티켓의 환불이 발생되는 경우 구매 페이지를 통해 환불 및 취소된 수량만큼 구매가 가능해 집니다.
이 경우 별도의 공지를 하지 않으니 티켓이 매진된 경우 수시로 구매 페이지를 확인하시면 환불 및 취소된 티켓을 구매하실 수 있습니다.

티켓 확정 시점

티켓은 구매 버튼을 누르는 것으로 확정되지 않습니다.
결제 완료 기준이며, 결제 진행 중 모든 티켓이 판매 완료되는 경우 결제 실패가 될 수 있습니다.

환불 규정

환불은 쿠팡 마이페이지에서 가능하며, 환불 요청 시 즉시 환불 혹은 결제 취소됩니다. (신용카드 결제의 경우 카드사에 따라 취소 완료가 지연될 수 있습니다.)
원활한 스프링 캠프 2018 진행을 위해서 해당 행사의 티켓은 4월 16일 14:00시에 구매가 완료된 모든 티켓에 대해 사용 처리 할 예정입니다. 4월 16일 14:00 이후에는 티켓 취소가 불가능하오니 이 점 참고하여 신중한 구매 및 취소 부탁드립니다.

4월 16일 14:00 이 후, 취소된 표를 구매하는 경우, 구매 익일에 사용처리가 될 예정입니다. 4월 16일14:00 이후에 취소표를 구매하는 분 역시 이 점 참조하여 신중한 구매 및 취소 부탁드립니다.

티켓 양도

구매 플랫폼을 통한 티켓 양도는 불가합니다.
티켓을 다른 분께 양도하시려는 경우 티켓 구매자분께서 스프링캠프 공식 이메일을 통해 양도 정보를 전달하셔야 합니다.
개인 간 티켓 양도에 따라 발생하는 문제에 대해서는 스프링캠프 및 쿠팡에서 해결해 드리지 않습니다.
